# 가장 중요한 원칙

Tesla 앱은

> **"내 차 상태"**

를 보여줍니다.

반면 FMS는

> **"관리자가 조치를 취해야 하는 정보"**

를 보여줘야 합니다.

즉,

```
데이터 표시
↓

문제 발견
↓

조치
```

가 UI의 흐름이 되어야 합니다.

# ① AI Summary (가장 위)

투자자에게도 가장 임팩트가 큽니다.

예를 들면

```
✓ 차량 정상

✓ 충전 필요 없음

✓ TPMS 정상

✓ 공조 ON

✓ 오늘 52km 운행

✓ 이상 징후 없음
```

만약 문제가 있으면

```
⚠ 배터리 18%

⚠ TPMS 후좌 낮음

⚠ 창문 열림

⚠ 충전 권장
```

이 카드 하나로

관리자가 바로 행동할 수 있습니다.

# 제가 BORI Fleet를 만든다면 추가할 "관리자 액션"

현재 UI는 **정보를 보여주는 것**에 집중되어 있습니다.

한 단계 더 나아가 **관리자가 바로 행동할 수 있는 버튼**을 상단에 배치하는 것을 추천합니다.

예를 들어:

* 🔒 잠금 / 잠금 해제
* ❄️ 공조 ON / OFF
* ⚡ 충전 시작 / 중지
* 📍 차량 위치 보기
* 📡 차량 깨우기(Wake Up)
* 🚨 경적 / 라이트 점멸(권한 있는 관리자만)

그리고 AI Summary 아래에는 **"권장 조치"**를 함께 표시합니다.

예시:

> **권장 조치**
>
> * 배터리 18% → 오늘 오후 6시 이전 충전 권장
> * TPMS 후좌 35 PSI → 점검 필요
> * 차량이 5일간 미운행 → 상태 확인 권장

이렇게 하면 BORI Fleet는 **Tesla 데이터를 보여주는 FMS**가 아니라, **관리자가 운영 결정을 내릴 수 있도록 도와주는 Fleet Intelligence Platform**으로 한 단계 더 차별화될 수 있습니다.
