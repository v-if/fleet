# 차량 상세 — 운영자 안내·액션 문구 (TRF-B1 정합) (VD-OPS)

| 항목 | 내용 |
|------|------|
| 목적 | Telemetry primary + **제원 Baseline(specs-only)** 기준으로 상세 화면 **안내문구·버튼명·액션 계층**을 맞춘다. SoC/위치는 Telemetry, 제원은 REST 1회 — 운영자가 헷갈리지 않게 한다 |
| 배경 | 「제원·상태 다시 불러오기」가 동적 상태까지 채우는 것처럼 읽힘. 신호 없음 배너가 제원 버튼을 동급으로 안내함. 새로고침과 제원 재조회의 차이가 불명확 |

## 1. 액션 3종의 역할 (현재 버전)

| 액션 | UI 위치(To-Be) | 하는 일 | SoC·위치 | 제원 |
|------|----------------|---------|:--------:|:----:|
| **새로고침** | 상단 툴바 | DB·목록 재조회 (+ registry sync). Tesla `vehicle_data` 없음 | ❌ | ❌ (이미 있는 값만 표시) |
| **실시간 연동 다시 켜기** | 신호 없음 배너 1순위 · 연동 끔 시 운영 | `fleet_telemetry_config` 재등록 → 스트림 | ✅ (수신 후) | ❌ |
| **제원 다시 불러오기** | 운영·제원 영역(2순위) | Baseline specs-only REST 1회 | ❌ | ✅ |

**원칙:** 신호 없음·관제 공백 → **재연동만** 1차 CTA. 제원 보정은 고급/운영. 새로고침은 “화면 다시 읽기”.

## 2. 문구 · 라벨 To-Be

### 2.1 버튼 라벨

| As-Is | To-Be |
|-------|-------|
| 제원·상태 다시 불러오기 | **제원 다시 불러오기** |
| 실시간 연동 다시 켜기 | 유지 |
| 새로고침 | 유지 |

### 2.2 신호 없음 배너 (`showConfigSyncedWarning`)

**As-Is:** 재연동 또는 제원·상태 다시 불러오기  

**To-Be:**
> 실시간 연동은 켜져 있지만 차량 신호가 아직 없습니다. SoC·위치 등 관제 데이터를 받으려면 「실시간 연동 다시 켜기」로 Telemetry 구독을 재등록하세요. 차량이 깨어 있어야 수신이 시작되기 쉽습니다.

배너 버튼: **재연동만** (+ 선택: 운영 액션 앵커). 제원 버튼 미노출.

### 2.3 lifecycle `TELEMETRY_PENDING`

**To-Be body:**
> 키가 확인되었습니다. 첫 실시간 신호를 기다리거나 「실시간 연동 다시 켜기」로 구독을 확인하세요. 제원만 비어 있으면 「제원 다시 불러오기」를 사용할 수 있습니다.

**To-Be actions:** `["reconnect_telemetry", "retry_baseline"]` (재연동 우선 표시)

### 2.4 성공·실패 토스트

| 상황 | To-Be |
|------|--------|
| 제원 성공 | 제원을 다시 불러왔습니다. (배터리·위치는 변경되지 않습니다.) |
| 제원 실패 | 제원 불러오기에 실패했습니다. 차량이 깨어 있을 때 다시 시도하세요. |
| Timeline Baseline | 제원 수집 (초기 REST) |

### 2.5 인근충전소 공백 안내

**As-Is:** 「제원·상태 다시 불러오기」로 갱신  

**To-Be:** (주차 후 Telemetry/정책에 맡김)
> 주차 후 갱신됩니다. (제원 다시 불러오기로는 인근충전소가 갱신되지 않습니다.)

또는 제원 언급 삭제:
> 주행 중 — 주차 후 갱신

### 2.6 운영 섹션 보조 문구

제원 버튼 근처:
> 모델·색상·소프트웨어 버전 등 고정 제원만 가져옵니다.
