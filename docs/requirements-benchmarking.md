# UMOS ONE Pleos Fleet 벤치마킹

## 1. 벤치마킹 개요

| 항목 | 내용 |
|------|------|
| 참조 자료 | `benchmarking/UMOS-ONE.pdf`, `benchmarking/UMOS-ONE-Pleos-Fleet.pdf` |
| 벤치마킹 대상 | UMOS ONE — **Pleos Fleet** (차량 관제 서비스) |
| 우리 프로젝트 | 1인 개발 MVP FMS (PC 웹 대시보드 중심) |
| 벤치마킹 목적 | 데모데이 수준 MVP에서 참고·차별화할 기능과 우선순위 도출 |

UMOS ONE은 모빌리티(TAP!), 물류(Capora TMS/WMS), 차량 관제(Pleos Fleet)를 아우르는 **Full-stack 통합 플랫폼**을 지향한다. Pleos Fleet은 그중 차량 관제 축이며, **Device & Deviceless** 방식으로 차량 데이터를 수집하고 실시간 관제·자동화·예지 정비까지 제공한다.

---

## 2. UMOS ONE / Pleos Fleet 핵심 포지셔닝

### 2.1 해결하려는 문제
- **파편화된 시스템**: Fleet·노선·화물이 통합 없이 분리 운영
- **운영 사각지대**: 실시간 데이터 부재로 사전 대응 불가
- **확장 불가 인프라**: 레거시 시스템이 자율주행 전환 속도를 따라가지 못함

### 2.2 Pleos Fleet 사용자 Pain Point (역할별)
| 역할 | 주요 불편 |
|------|-----------|
| 기업 관리자 | 운행 일지 작성 부담, 차량 사용 현황 파악 어려움, 유지비 절감 필요 |
| 차량 관리자 | 점검 우선순위 불명확, 정비 이력 분산, 부품 교체 시기 놓침 |
| 운전자 | 위치 문의 부담, 경고등·사고 불안, 차량 예약 충돌 |

### 2.3 Pleos Fleet 제공 가치
- 실시간 운행 이슈 파악 → 운영 효율
- 운행 이력·에너지 효율(전비) 분석 → 성과 평가 (원문은 연료 효율, 우리 타깃은 EV이므로 전비 기준)
- 차량 문제 예방 → 수리 비용 절감
- 정비 이력 투명화 → 잔존 가치 상승
- 안전 운전 분석 → 사고 예방·보험료 절감

## 3. Pleos Fleet 기능 맵 (벤치마킹 기준)

### 3.1 실시간 관제
| 기능 | 설명 |
|------|------|
| 실시간 위치·경로 | 이동 경로 및 실시간 위치 가시화 |
| 차량 상태 모니터링 | 이상상태, 경고등, 충전·배터리, 시동 ON/OFF |
| 상황 관제 | 도난, 사고, 공회전 모니터링 |
| EV 정보 | 충전 상태, 배터리, 주행가능 거리, 연비 |
| 차량 온도 관제 | 냉동기 온도, 목표 온도, Door ON/OFF (냉동차 대상) |

### 3.2 대시보드
- 전체 차량 가동률
- 이상 상태 / 장기 미운행 차량 조회
- 주행 거리별 차량 조회
- 안전 운전 현황
- 총 운행거리

### 3.3 차량·운행 관리
| 영역 | 기능 |
|------|------|
| 차량 기본정보 | 차량번호, 차종, 연식 |
| 차량 운행 정보 | 실시간 위치·이상상태, 이동거리, 최신 경로, 장기 미운행 |
| 스케줄 관리 | 일정별 사용형태·사용자, 차량 이용 상태 |
| 운행 이력 | 기간별 동선, 이상 발생 시점, 사고 관리 |
| 운행일지 | 이동 거리·날짜 기반 자동 생성, 국세청 제출용 운행기록부 |
| e-TAS | 운행 기록 자동 전송 |

### 3.4 지오펜싱·관심 지역
- Geofencing 기반 관심 지역 설정
- 진출입 이력 자동 기록
- 차량별 관심지점 체류 시간 조회

### 3.5 안전 운전·운전자 관리
- 위험 운전: 과속, 급출발, 급가속, 급감속
- 발생 위치·시간·횟수, 구간별 속도 시각화
- 운전자별 안전 운전 점수 (주행거리·시간 반영)
- 운전자 정보·운행 이력 관리

### 3.6 정비·예지 정비
- 정비 이력 통합 관리
- 정비 예약·진행·완료 상태 모니터링
- 기아 Auto Q 연동 정비 예약
- **EV 배터리 Health 예측** (충전/운전 패턴 AI 분석)
- **타이어 마모도** 실시간 모니터링·교체 시기 제안

### 3.7 차량 원격 제어
- 공조기 ON/OFF, Door Lock/Unlock
- 충전 시작/종료 예약
- 단일·복수 차량 온도 설정 (냉동기)
- 차량 제어 요청·실행·결과 이력

### 3.8 데이터 수집 방식
- **Device & Deviceless Fleet Management**: 단말기 유·무 관계없이 차량 통합 관제

## 4. 우리 MVP와의 비교

### 4.1 이미 요구사항에 포함된 영역 (벤치마킹 정합)
| 우리 MVP (`requirements.md`) | Pleos Fleet 대응 | 벤치마킹 시 참고 포인트 |
|------------------------------|-------------------|------------------------|
| 차량 목록·상세 조회 | 차량별 상태 정보, 기본정보 | 목록에 상태 배지(온라인/이상/미운행) 표시 |
| 위치 조회 | 실시간 위치·경로 | 지도 + 차량 마커, 최신 경로 표시 |
| 차량 상태 정보 | 이상상태, 배터리, 경고등 | EV 타깃이면 배터리·주행가능거리 우선 |
| 전체 현황 대시보드 | 가동률, 이상차량, 미운행 | KPI 카드 3~4개로 요약 (과도한 위젯 지양) |
| 비정상 상태 식별 | 이상상태, 경고등 관제 | 대시보드 상단 알림·필터로 강조 |
| 최근 데이터 수신 시간 | 실시간 정보 | “마지막 업데이트” 타임스탬프 필수 |
| 소프트웨어 기반 데이터 수집 | Deviceless 방향 | 단말기 없이 OEM/API·앱 연동 전략과 일치 |

### 4.2 MVP에서 참고할 만한 추가 아이디어 (우선순위별)

#### P0 — 데모 임팩트 대비 구현 난이도 낮음
| 기능 | Pleos Fleet 참고 | MVP 적용 제안 |
|------|------------------|---------------|
| 이상/미운행 차량 필터 | 장기 미운행 차량 관리 | N일 이상 미운행 차량 목록·배지 |
| 차량 상세 KPI | 실시간 차량·주행 정보 패널 | 위치, 배터리, 시동, 최종 업데이트 한 화면 |
| 운행 거리 요약 | 총 운행거리, 기간별 집계 | 일/주 단위 주행거리만 우선 |
| 상태 색상 코딩 | 이상상태 즉시 파악 | 정상/주의/이상 3단계 UI |

#### P1 — 데모 스토리 강화 (중기)
| 기능 | Pleos Fleet 참고 | MVP 적용 제안 |
|------|------------------|---------------|
| 운행 이력·동선 | 기간별 동선 확인 | 날짜 선택 → 지도에 경로 재생 |
| Geofencing | 관심지역 진출입 | 관심 지점 1~2개, 진입 알림만 |
| 안전 운전 요약 | 위험 운전 횟수 | 급가속/급감속 카운트만 (점수화는 후순위) |
| 운행일지 자동 생성 | 거리·날짜 기반 | CSV/PDF 간단 export |

#### P2 — 투자·확장 단계 (MVP 이후)
| 기능 | Pleos Fleet 참고 | 비고 |
|------|------------------|------|
| 차량 원격 제어 | 공조, 도어, 충전 예약 | OEM API·권한·안전 이슈로 후순위 |
| 정비 이력·예약 | Auto Q 연동 | 제조사 파트너십 필요 |
| EV 배터리 Health 예측 | AI 수명 예측 | 데이터 축적 후 |
| 타이어 마모 예측 | 운전습관+도로 데이터 | 센서·모델 의존 |
| e-TAS / 국세청 운행기록부 | 규제 대응 | B2B 요구 발생 시 |
| 운전자 점수·보험 연계 | 안전 운전 가이드 | 운전자 앱·동의 체계 필요 |
| 냉동차 온도 관제 | 냉동기·Door 제어 | 타깃 차종·산업 다를 때 |

### 4.3 의도적으로 벤치마킹하지 않을 영역 (MVP 제외)
- AI Agent 기반 운영 자동화·챗봇
- 온톨로지 기반 의사결정
- TMS/WMS·물류 통합
- 자율주행·로보트럭 연계
- 스마트시티 Command Hub
- 다역할(기업·정비·운전자) 풀 포털
- 복수 차종·복수 OEM 동시 지원

## 5. 화면·UX 벤치마킹

Pleos Fleet 주요 화면 구성을 MVP 웹 대시보드 설계 시 참고한다.

| 화면 | Pleos Fleet 구성 | MVP 권장 |
|------|------------------|----------|
| 메인 대시보드 | 가동률, 이상/미운행, 주행거리, 안전현황 | 좌: KPI 카드 / 우: 지도 또는 차량 리스트 |
| 차량 목록 | 전체 목록 + 스케줄 연동 상태 | 검색·필터(상태, 미운행) |
| 차량 상세 | 실시간 정보 + 주행 + 제어 | 실시간 상태 + 최근 경로 (제어 탭 제외) |
| 운행 이력 | 기간별 시간/거리, 동선 | 단일 차량·단일 날짜 뷰로 시작 |
| 알림/이상 | 경고등, 이상상태 | 대시보드 알림 패널 |

**UX 원칙 (Pleos에서 차용)**
- “모든 차량을 하나의 시선으로” — 한 화면에서 전체 현황 파악
- 문제 차량을 먼저 보여주는 **관제자 중심** 정보 구조
- 이력·상태·위치를 영역별로 분리하되, 상세 진입은 1~2클릭 이내

## 7. 차별화·포지셔닝 시사점

UMOS ONE은 **대기업급 Full-stack + AI + 자율주행** 스토리다. 1인 MVP는 다음으로 차별화·집중할 수 있다.

| UMOS ONE 강점 | 우리 MVP 대응 전략 |
|---------------|-------------------|
| 플랫폼 통합 (TMS/WMS/셔틀) | **차량 관제 단일 제품**에 집중, 데모 메시지 단순화 |
| AI·예지 정비 | **실시간 가시성**과 **이상 탐지**에 먼저 승부 |
| 다역할·다기능 | **관리자 웹 1 persona**만 완성도 있게 |
| Device+Deviceless | **Deviceless + 전기차 1종**으로 빠른 온보딩 스토리 |
| 엔터프라이즈 연동 (e-TAS, Auto Q) | 데모용 **합성/샘플 데이터 + 실제 API 1종** |

데모데이 메시지 예시:
> “전용 단말기 없이 전기차 데이터만 연결해도, 관리자는 한 화면에서 전체 차량 위치·상태·이상 차량을 즉시 확인할 수 있다.”
