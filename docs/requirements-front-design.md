# FMS 프론트엔드 디자인 요구사항 정의서

## 1. 문서 개요

| 항목 | 내용 |
|------|------|
| 목적 | Pleos Fleet를 벤치마킹하되, **테슬라·EV 오너에게 훨씬 직관적이고 매력적인 UI**로 차별화하는 프론트 디자인 방향 정의 |
| 참조 이미지 | `benchmarking/dashboard.png`, `benchmarking/vehicles.png`, `benchmarking/vehicles-info.png` |
| 관련 문서 | `requirements.md`, `requirements-benchmarking.md`, `requirements-tesla-api.md`, `development-checklist.md` |
| 적용 범위 | Phase 2.2 UI/UX 개선 (Mock 데이터 기반, 실 연동은 Phase 3 이후) |

> 벤치마킹 대상은 UMOS ONE **Pleos Fleet** PC 웹 관제 화면이다. 단, 우리는 Pleos를 **그대로 따라가지 않는다.** Pleos의 검증된 정보 구조(카드 위젯·상태 색상·상세 헤더)는 취하되, **비주얼·인터랙션은 테슬라 오너 감성에 맞춰 완전히 재해석**한다.
>
> **한 줄 요약**: 유모스원이 *모든 상용차를 위한 무겁고 거대한 정장*이라면, 우리는 *테슬라 오너의 취향을 저격하는 핏하고 스마트한 스포츠웨어*다.

## 2. 핵심 디자인 전략 & 포지셔닝

### 2.1 포지셔닝 — "정장 vs 스포츠웨어"

| 구분 | UMOS ONE Pleos Fleet | **우리 FMS** |
|------|----------------------|--------------|
| 비유 | 모든 상용차를 위한 **무겁고 거대한 정장** | 테슬라 오너 취향을 저격하는 **핏하고 스마트한 스포츠웨어** |
| 타깃 | 대규모·다차종 상용 플릿 (수백~수천 대) | **테슬라·EV 오너 / 소규모 플릿 (1~10대)** |
| 정보 밀도 | 최대 밀도 — 모든 기능을 한 화면에 | **선택과 집중** — 핵심만, 여백과 리듬 |
| 시각 톤 | 라이트 엔터프라이즈, 기능 우선 | **모던·프리미엄·EV 감성** (테슬라 앱 톤) |
| 메뉴 | 10개+ 다기능 포털 | 3~4개 핵심, 군더더기 없음 |
| 첫인상 | "많다 / 전문적이다" | **"예쁘다 / 갖고 싶다 / 직관적이다"** |

### 2.2 전략 3대 축

1. **테슬라 오너 취향 저격 (Aesthetic-first)**
   - 테슬라 모바일 앱·차량 UI를 연상시키는 **미니멀·프리미엄** 비주얼.
   - 기능 나열이 아니라 **감성적 만족(갖고 싶은 대시보드)** 을 우선.

2. **지도 중심의 "직관 + 미감" (Map as Hero)**
   - 데모데이 규모는 **1~10대**. 표에 나열하는 방식은 Pleos가 이미 잘한다.
   - 우리는 **소수 차량이 지도 위에서 얼마나 직관적이고 이쁘게 표현되는가**에 집중한다.
   - 지도는 보조 위젯이 아니라 **주인공(Hero)** 이다.

3. **경량·스마트 (Lean & Smart)**
   - Pleos의 무거운 포털 대신 **빠르고 가벼운** 인터랙션.
   - 불필요한 기능·컬럼·메뉴를 과감히 제거 → "핏(fit)한" 경험.

### 2.3 데모데이 성공 이미지

> "심사위원이 화면을 보는 순간, **'테슬라 대시보드 같다 / 예쁘다'** 는 반응이 먼저 나오고, 지도 위 소수 차량의 상태(위치·배터리·이상)를 **설명 없이도** 이해한다."

- 핵심 KPI: **차량 수**가 아니라 **첫인상의 매력도 + 지도의 직관성**.
- 10대 이하를 **아름답게** 보여주는 것이, 1000대를 빽빽하게 보여주는 것보다 강력하다.

### 2.4 차별화 원칙 (Pleos 대비 하지 말 것)

| 하지 말 것 (Pleos식) | 대신 (우리식) |
|----------------------|---------------|
| 모든 위젯을 한 화면에 욱여넣기 | **핵심 위젯 + 지도 Hero** 로 정제 |
| 촘촘한 다컬럼 테이블에만 의존 | 지도·카드·시각화 **그래픽 우선** |
| 10개+ 사이드바 메뉴 | 3~4개 + 명확한 활성 상태 |
| 무채색 엔터프라이즈 톤 | **EV 프리미엄 톤**(딥 톤 + 포인트 액센트, 부드러운 모션) |
| 기능 과시 | **첫인상·직관성 과시** |

## 5. 디자인 원칙 (FMS 적용)

### 5.1 채택할 원칙 (구조 — Pleos에서 취함)
1. **카드 기반 위젯**: 대시보드·상세 모두 Card 단위로 정보 구획
2. **색상 계층**: 정상(녹/기본) · 주의(amber) · 이상(red) · 정보(blue/secondary)
3. **스캔 가능성**: 숫자 KPI + 이상 차량 **리스트+태그** 병행
4. **breadcrumb + 페이지 컨텍스트**: 어디에 있는지 항상 표시
5. **마지막 업데이트 + 새로고침**: 관제 신뢰감 (데이터 신선도)
6. **시각화 우선**: TPMS·배터리·지도는 텍스트보다 도식/게이지

### 5.2 비주얼 언어 (감성 — 테슬라 오너 저격, §2 전략의 핵심)
1. **딥 톤 베이스 + 포인트 액센트**
   - 다크/딥 그레이 기반에 **테슬라 레드 또는 EV 그린·블루** 포인트 1색.
   - 상태 색(정상/주의/이상)은 유지하되 **채도·명도를 세련되게** 조정.
2. **넉넉한 여백과 리듬**: 정보를 욱여넣지 않고, "핏(fit)"한 간격·타이포 위계.
3. **부드러운 모션 (Micro-interaction)**
   - 카드 hover, 마커 등장/선택, 숫자 카운트업, 새로고침 스핀 등 **가벼운 전환**.
   - 과하지 않게 — 프리미엄은 절제에서 나온다.
4. **둥근 모서리 + 부드러운 그림자 / 글래스 느낌** (선택): 테슬라 앱스러운 카드.
5. **아이콘·타이포**: lucide 아이콘 + 명료한 sans-serif, 숫자는 굵고 크게(Hero 숫자).
6. **다크 모드 우선 고려** (P1): EV 오너 감성과 지도 대비에 유리.

### 5.3 지도 Hero 원칙 (데모의 승부처, §2.2)
- **소수(1~10대)를 아름답게**: 마커는 테슬라 차량 느낌의 **커스텀 마커/핀** (기본 물방울 지양).
- 마커에 **배터리·상태 색**을 담아 한눈에 파악 (예: 링/배지형 마커).
- 선택 시 **부드러운 포커스**(줌·하이라이트) + 미니 요약 카드.
- 지도 스타일도 **톤에 맞게**(라이트/다크 지도 테마 검토), 마커 밀집 시 정돈.
- 데모 시나리오: "지도만 봐도 어느 차가 이상인지 즉시 안다."

### 5.4 MVP에서 제외·축소
- Pleos 전체 사이드바 메뉴 (업무·정비·사고·기사 등) → 3~4개 핵심만
- 스케줄·안전운전 점수·관심지역·실시간 영상 탭
- 플릿 멀티셀렉트·다운로드·컬럼 설정 (P2)
- 주행 거리 차트·Top 5 (데이터 없으면 Mock도 부담)
- **과밀 정보**: Pleos식 최대 밀도는 의도적으로 **버린다** (§2.4)

### 5.5 브랜딩
- 제품명: **Fleet FMS** (사이드바·title) — 향후 감성 네이밍 검토 여지
- 포지셔닝: **테슬라 오너를 위한 스마트 스포츠웨어** (경량·프리미엄·직관)
- shadcn/ui + Tailwind 기반, **다크·딥 톤 사이드바 권장**(P1) — 첫인상 차별화 포인트

### 6.2 대시보드 (`/`)

#### 목표
**지도를 Hero로** 둔 관제 홈. 소수 차량을 지도 위에서 아름답고 직관적으로 보여주고, 이상·미운행은 곁들이는 위젯으로. (Pleos식 위젯 나열이 아닌 **지도 중심 재구성**)

| ID | 요구사항 | Pleos 참고 | 우선순위 |
|----|----------|------------|:---:|
| DSH-00 | **지도 Hero**: 상단에 크게 배치, 커스텀 마커(상태·배터리 색), 부드러운 진입 모션 (§5.3) | — (차별화) | **P0** |
| DSH-01 | KPI 카드 행 (전체·정상·충전중·주의·이상·미운행) — Hero 숫자 스타일 | 숫자 요약 | P0 |
| DSH-02 | **이상 상태 차량** 위젯: 연결(오프라인)·운행(주의)·차량(이상) **건수** + 차량 리스트 + **이슈 태그** | 이상 상태 차량 카드 | P0 |
| DSH-03 | **장기 미운행** 위젯: N일(7/30) 선택 + 차량별 **기간·위치(주소 또는 좌표)** | 장기 미운행 카드 | P0 |
| DSH-05 | 위젯 카드 **더보기** → 차량 목록(필터 적용) 또는 상세 링크 | > 링크 | P1 |
| DSH-06 | 지도 마커 ↔ 위젯/목록 **연동 하이라이트** (마커 선택 시 요약) | — | P1 |
| DSH-07 | 주행 거리 차트·Top 5·관심 지역·안전운전 위젯 | 해당 카드들 | P2 |

#### 레이아웃 권장 (데스크톱) — 지도 Hero
```
[KPI 6열 — 컴팩트]
[            실시간 지도 (Hero, 넓게)            ]
[이상 상태 차량 (1/2)]     [장기 미운행 (1/2)]
[충전중/배터리 낮음 요약 (전체)]                  ← P1
```
> Pleos는 지도가 여러 위젯 중 하나였지만, 우리는 **지도를 화면의 주인공**으로 끌어올린다. (데모 규모 1~10대에 최적)

## 8. 우선순위 요약 (Phase 2.2 범위)

### P0 — 데모 필수 UI (차별화 핵심 포함)
- **지도 Hero + 커스텀 마커** (§5.3 / DSH-00) — 데모 승부처
- breadcrumb, 새로고침, 페이지 배지(총 N건)
- **테슬라 톤 비주얼 언어 적용** (딥 톤 + 포인트, 여백/타이포) (§5.2)
- 대시보드: 이상 상태·미운행 위젯 + 지도 중심 레이아웃
- 목록: 헤더·필터·컬럼 정리
- 상세: 요약 헤더 + 탭(홈/이벤트) + 경고 태그 + 홈 탭 레이아웃

### P1 — 데모 강화
- 마이크로 인터랙션(모션 토큰 §7.3), 다크/딥 톤 사이드바
- TPMS 도식, 배터리 게이지, KPI 카운트업
- 마커 ↔ 위젯 연동, 위젯 더보기, 페이지네이션
- 상세 푸터·액션 버튼 placeholder

### P2 — 투자 이후
- 차트 위젯, 다운로드, 추가 탭 본문, 글로벌 헤더, 지도 테마 커스텀

## 9. 완료 기준 (Phase 2.2)

1. **첫인상 테스트**: 화면을 처음 본 사람이 "예쁘다/테슬라 같다"고 느끼고, 지도만으로 이상 차량을 식별 (§2.3)
2. **지도 Hero**: 1~10대가 커스텀 마커로 직관적·매력적으로 표현됨 (§5.3)
3. Pleos 핵심 UX 구조(카드 위젯·breadcrumb·상세 헤더+탭·경고 태그)가 Mock으로 동작
4. 대시보드에서 이상·미운행을 **건수+목록+태그**로 확인 가능
5. 차량 상세에서 **한 눈에 상태 파악**(요약 헤더 + 홈 탭 시각화)
6. `pnpm lint`, `pnpm build` 통과 / Phase 2·2.1 기능 회귀 없음
