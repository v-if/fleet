# VD3 상세 상단 — PC·모바일 툴바 통일 (VD3-R)

| 항목 | 내용 |
|------|------|
| 목적 | `/vehicles/[id]` **기본 차량상세** 상단에서 PC·모바일 **표시명(`plateNumber`)과 [새로고침]을 같은 행**에 두고, 컷오버 후 불필요해진 **「이전 상세 (v2)」진입 링크를 화면에서 제거**한다 |
| 배경 | PC는 `FleetToolbar`가 `lg:flex-row`라 제목·새로고침이 한 줄. 모바일은 `flex-col`이라 제목 → 새로고침이 줄바꿈되고, 그 아래 **목록으로 · 이전 상세 (v2)** 행이 이어져 **3단 스택**처럼 보인다. VD3 컷오버(VD3-7) 이후 일상 관제는 기본 상세만 쓰므로 v2 링크는 상단 CTA로 둘 필요가 없다 |
| 관련 | [requirements-vehicle-detail-vd3.md](./requirements-vehicle-detail-vd3.md) (컷오버), [checklist-vehicle-detail-vd3.md](./checklist-vehicle-detail-vd3.md) |
| 적용 | **기본 상세만** (`FleetVehicleDetailViewV3` · `/vehicles/[id]`). `/vehicles/[id]/v2` 화면·URL은 **유지** (직접 접속·롤백용) |
| 상태 | **코드 ✅ (VD3-R-2) · 검수 VD3-R-3 ☐ · 식별 표시는 VL-P-5로 Hero만** |
| 작성일 | 2026-07-16 |
| ID | **VD3-R** |

## 1. As-Is 분석

### 1.1 레이아웃 원인

| 요소 | 코드 | 모바일 | PC (`lg+`) |
|------|------|--------|------------|
| `FleetToolbar` | `flex-col` → `lg:flex-row` | title / 새로고침 **세로 분리** | title · 새로고침 **한 행** |
| 상세 하단 링크 행 | `목록으로` + `이전 상세 (v2)` | 툴바 아래 **추가 행** | 동일 |

요청에서 본 모바일 스택:

```text
plateNumber
[새로고침]
[목록으로]
[이전 상세 (v2)]   ← 제거 대상
```

원하는 형태 (PC와 동일 원칙):

```text
plateNumber                    [새로고침]
[목록으로]                     ← 2행 허용 (네비게이션)
```

### 1.2 v2 링크 위치 (기본 상세)

| 위치 | 현재 |
|------|------|
| 툴바 바로 아래 링크 행 | 「이전 상세 (v2)」 |
| 운영 섹션 | 「이전 상세 (v2)」 |

→ **둘 다 UI에서 제거** (요청: 「이전 상세(v2) 제거」). URL `/vehicles/{id}/v2`는 북마크·개발자 직접 진입용으로 **라우트만 유지**.

### 1.3 `FleetToolbar` 공유 범위

`FleetToolbar`는 대시보드·목록·지도·설정에도 쓰인다.  
본 요구는 **차량상세** 통일 — **전역 `lg:` 브레이크포인트를 무조건 제거하면 다른 화면에도 영향**.

## 2. 제안 분석

### 2.1 채택

| 점 | 이유 |
|----|------|
| 표시명 + 새로고침 동일 행 (전 폭) | 관제 1차 식별·갱신이 한눈에 · 모바일 세로 낭비 감소 |
| 「이전 상세 (v2)」제거 | 컷오버 후 일상 CTA가 아님 · 혼동·스크롤 낭비 |
| `/v2` 라우트 유지 | 완전 삭제가 아니라 「링크만 숨김」 — 롤백·비교 가능 |

### 2.2 빈칸 보완

| 공백 | 보완 |
|------|------|
| 「목록으로」처리 | **유지**. 새로고침과 같은 행에 넣지 않음 — 모바일에서 제목 폭이 줄어듦. **툴바 아래 단독 행** 또는 툴바 `actions`에 두되 제목 행과는 분리 |
| 긴 표시명 | 제목 `truncate` / `min-w-0` · 새로고침 `shrink-0` — `영업1팀`·`TESLA-214742` 모두 한 행 유지 |
| 구현 방식 | **권고 A:** `FleetToolbar`에 `layout="inline"` (또는 `alwaysRow`) prop — 상세만 사용. **대안 B:** 상세 전용 헤더(툴바 복제) — 중복↑ → 비권고 |
| 전역 영향 | 대시보드·목록 툴바는 **기존 `lg:flex-row` 유지** (변경 없음) |
| v2 화면 | `/v2` 내부의 「현재 상세」링크는 **유지** (이전 화면에서 기본으로 돌아가기) |
| 운영 섹션 v2 링크 | 상단과 동일하게 **제거** |

### 2.3 비범위

- `/v2` 라우트·`FleetVehicleDetailView` 삭제  
- 대시보드·목록·지도·설정 `FleetToolbar` 모바일 레이아웃 전면 변경  
- Hero 카드 내부 표시명(연필) 레이아웃 (VD3-N — 별개)  
- 모바일 전용 하단 탭바·앱 셸

## 3. To-Be UI

### 3.1 상단 (기본 상세)

```text
┌─────────────────────────────────────────────┐
│ {plateNumber}              [새로고침]       │  ← 모든 viewport 동일 행
│ {model · year}                              │  ← description (있을 때)
├─────────────────────────────────────────────┤
│ [목록으로]                                  │  ← 네비만 · v2 링크 없음
└─────────────────────────────────────────────┘
```

### 3.2 동작·스타일

| 항목 | 규칙 |
|------|------|
| 제목·새로고침 | `flex-row items-center justify-between` (상세용 layout) · 모바일에서도 세로 스택 금지 |
| 제목 | `truncate` · 길면 말줄임 · `title` attr로 전체 표시명 |
| 새로고침 | 기존 문구·disabled 동작 유지 · `shrink-0` |
| 목록으로 | 유지 · v2 링크 삭제 |
| 운영 섹션 | 「이전 상세 (v2)」버튼/링크 삭제 |

### 3.3 `/v2` 접근

| 방식 | 상태 |
|------|------|
| UI 링크 (기본 상세) | ❌ 제거 |
| URL 직접 `/vehicles/{id}/v2` | ✅ 유지 |
| v2 → 「현재 상세」 | ✅ 유지 |

## 4. 수용 기준

1. 모바일·PC 모두 상세 상단에서 **표시명과 [새로고침]이 한 행**.
2. 기본 상세 어디에도 **「이전 상세 (v2)」문구/링크 없음** (상단·운영).
3. **[목록으로]** 로 `/vehicles` 이동 가능.
4. `/vehicles/{id}/v2` 직접 접속 시 이전 상세 정상 · 「현재 상세」로 기본 복귀 가능.
5. 대시보드·목록·지도 툴바 모바일 레이아웃 **회귀 없음** (기존과 동일).
6. 긴 표시명에서도 새로고침 버튼이 줄바꿈으로 밀려 내려가지 않음 (말줄임으로 처리).

## 6. 의견 · 진행 여부

### 판단: **GO**

| 근거 | 설명 |
|------|------|
| 제품 | 컷오버 직후 모바일 상단이 어수선함 — **식별+갱신**을 한 행으로 맞추는 것이 관제 UX에 맞음 |
| 리스크 | UI만 · API/스키마 없음. 상세 한정 prop이면 타 화면 회귀 최소 |
| 범위 | v2 **링크 제거 ≠ 기능 삭제** — 요청과 정합 |

### 주의

- 「PC·모바일 통일」을 **화면 전체 리디자인**으로 확대하지 말 것 — 본 Phase는 **상단 툴바·네비 행**만.
- `FleetToolbar` 전역 `flex-col` 삭제는 목록/대시보드에서 제목이 긴 경우 버튼과 충돌할 수 있음 → **상세 opt-in** 권고.

**추천:** R-1 승인 후 R-2 즉시 구현 (반나절 이내 규모).
