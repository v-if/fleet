# 보리차 IR 초안

스마일게이트 오렌지플래닛 2026 하반기 정기모집 제출용 초안이다.  
현재까지 개발된 소스코드와 `docs/` 문서를 근거로 작성했으며, 실제 수치 검증이 필요한 항목은 별도 코멘트로 남긴다.

## 1. 표지 및 회사 목적
### 서비스명
보리차

### 한 줄 소개
전용 단말기 없이 공식 OEM API만으로 시작하는 법인 차량 운영 관제 SaaS

### 비전
보리차는 법인 차량 운영을 엑셀, 전화, 카카오톡, 수기 운행일지 중심의 아날로그 관리에서 벗어나 데이터 기반 운영 체계로 전환하는 것을 목표로 한다. 초기에는 테슬라 단일 차종으로 빠르게 검증하고, 이후 다양한 OEM과 운영 환경을 연결하는 표준형 차량 운영 인프라로 확장한다.

### 엘리베이터 피치
보리차는 법인 차량 운영 담당자가 여러 차량의 위치, 배터리, 충전 상태, 이상 여부를 한 화면에서 확인하고 조치할 수 있게 해주는 차량 관제 SaaS다. 별도 OBD/IoT 단말기 설치 없이 제조사 공식 API와 OAuth 기반으로 데이터를 수집하고, 실데이터 연동이 어려운 상황에서도 Mock 폴백과 가상 차량 시드 기능으로 안정적인 데모와 PoC를 가능하게 한다.

## 2. 문제 정의
법인 차량 운영 현장은 여전히 사람 중심의 비효율이 크다. 차량이 적을 때는 버틸 수 있지만, 차량 수가 늘어나면 운영 가시성과 통제가 급격히 떨어진다.

### 현장의 주요 페인 포인트
- 누가 어떤 차량을 언제 운전했는지 추적이 어렵다.
- 사고 발생 시 책임 소재가 모호하다.
- 운행일지 작성, 주유 영수증 정리, 출장 후 운행기록부 작성 같은 반복 행정 업무가 번거롭다.
- 검사, 보험, 정비, 배터리 상태 같은 유지관리 항목을 놓치기 쉽다.
- 영업회사, AS 조직, 렌터카 등은 차량이 지금 어디 있는지, 언제 회수 가능한지 즉시 파악해야 한다.
- 전기차 비중이 높아질수록 충전 상태, SOC, 충전 습관 관리가 새로운 운영 부담이 된다.
- 관리자 입장에서는 엑셀, 전화, 메신저, 운행일지, 보험, 정비, 검사, 주유를 모두 사람이 직접 연결해 관리하는 경우가 많다.

### 기존 대안의 한계
- 제조사 앱은 차량 단위 확인에는 유용하지만, 관리자 관점의 통합 운영 화면을 제공하지 못한다.
- 기존 플릿 솔루션은 도입 장벽이 높거나 특정 환경에 최적화된 경우가 많다.
- 전용 하드웨어 설치형 솔루션은 초기 도입 비용과 현장 운영 부담이 크다.

## 3. 해결책
보리차는 법인 차량 운영의 핵심 문제를 해결하기 위해 관리자 중심의 PC 웹 대시보드를 제공한다.

### 핵심 가치 제안
- 전용 단말기 없이 공식 OEM API 기반으로 차량 데이터 수집
- 차량 목록, 상세, 지도, 충전 상태, 이상/주의 차량을 한 화면에서 확인
- 실데이터 연동 실패 시 Mock 폴백으로 데모/PoC 안정성 확보
- 향후 운전자 관리, 배차 기록, 사고 관리, 차량 유지관리, 충전 관리 기능으로 확장 가능한 구조

### 현재 구현된 솔루션 범위
- Tesla OAuth 연동
- Tesla Fleet API 기반 차량 데이터 수집
- 차량 목록/대시보드/지도/차량 상세/연동 설정 화면
- 실데이터와 Mock 데이터 전환 구조
- 가상 차량 추가 기능
- AuditLog / ApiCallLog 기반 감사 및 추적 구조
- Fleet Telemetry 전환을 위한 차세대 요구사항 설계 완료

## 4. 시장 규모 및 기회
보리차의 초기 시장은 전기차를 업무용으로 운영하는 국내 법인 플릿이다. 특히 전담 관제 시스템 구축이 부담스럽지만 차량 상태와 위치를 통합 관리해야 하는 중소 규모 조직을 우선 타깃으로 한다.

### 초기 타깃 고객
- 영업 차량 운영 기업
- 서비스/AS 방문 차량 운영 조직
- 공용 차량 운영 기업
- 소규모 렌터카/법인차 운영사
- EV 전환이 진행 중인 법인 고객

### 시장 기회
- 전기차 전환이 가속될수록 배터리·충전·운행 상태 관리 수요가 커진다.
- 공식 OEM API를 활용한 Deviceless 운영 도구는 초기 도입 장벽이 낮아 중소 규모 고객에 적합하다.
- 국내에서 PMF를 검증한 뒤, OEM API 접근성이 높고 EV 운영 수요가 큰 해외 시장, 특히 북미로 확장할 수 있다.

### TAM / SAM / SOM 작성 방향
- TAM: 법인 차량 운영 SaaS 및 플릿 관리 시장
- SAM: 국내 EV 법인 플릿 운영 시장
- SOM: 국내 테슬라 및 초기 EV 법인 고객 중 보리차가 1~3년 내 직접 공략 가능한 세그먼트

## 5. 제품/서비스 소개
보리차는 현재 관리자 중심의 웹 기반 차량 운영 콘솔 형태로 구현되어 있다.

### 사용 흐름
1. 사용자가 로그인한다.
2. Tesla 계정을 OAuth로 연결한다.
3. `/settings`에서 연동 상태와 동기화 상태를 확인한다.
4. `/api/sync/vehicles`를 통해 차량 데이터를 동기화한다.
5. 대시보드에서 전체 차량 현황, 이상/주의 차량, 장기 미운행, 충전 현황을 확인한다.
6. 차량 목록에서 전체 차량을 검색·필터링한다.
7. 차량 상세에서 위치, 배터리, 주행거리, TPMS, 잠금/문/창문, 공조, 정비 상태를 확인한다.

### 현재 화면 구성
- 대시보드: KPI 6카드, 실시간 지도, 이상/주의 차량, 충전 현황, 최근 차량
- 차량 목록: 상태/충전/위치 기반 목록 조회
- 지도: 전체 차량 위치 확인
- 차량 상세: 배터리, 위치, 상태, TPMS, 정비, 이벤트 확인
- 설정: Tesla OAuth 연결, 동기화, 폴백 상태 확인

### 데모 안정화 장치
보리차는 실제 차량이 sleep 상태이거나 API 응답이 희소한 상황을 대비해 `차량 추가(가상)` 기능을 구현했다. 이를 통해 실제 연동 없이도 1~5대의 가상 차량을 즉시 생성해 동일한 UI 흐름을 검증할 수 있다. 이는 단순한 데모 기능이 아니라 초기 세일즈와 PoC 과정에서 제품 신뢰도를 높이는 핵심 장치다.

## 6. 비즈니스 모델
기본 모델은 B2B SaaS 구독형이다.

### 과금 구조
- 기본 사용료: 법인 계정 단위
- 차량 과금: 차량 대수 구간별 추가 과금
- 상위 플랜: 실시간 Telemetry, 고급 알림, 리포트/다운로드, 운영 자동화 기능
- 엔터프라이즈 플랜: 구축 지원, 커스텀 연동, 멀티 OEM 통합

### 왜 이 모델인가
- 고객은 월 예산을 예측하기 쉽다.
- 차량 수 증가와 함께 고객 가치와 데이터 처리량이 함께 늘어난다.
- 초기에는 단순하고 설명 가능한 과금 구조가 유리하다.

### 향후 확장 수익원
- 실시간 Telemetry 기반 프리미엄 플랜
- 알림 정책/운전자 관리/배차 관리 모듈
- 운영 컨설팅 및 맞춤 연동 서비스

## 7. 경쟁사 분석 / 경쟁 우위
보리차의 경쟁 상대는 크게 두 범주다.
- 기존 플릿 솔루션
- 수기 관리 + 제조사 앱 개별 확인

### 보리차의 경쟁 우위
1. 공식 OEM API 기반 Deviceless 접근  
별도 OBD나 IoT 단말기 설치 없이도 제조사 공식 API와 OAuth 기반으로 시작할 수 있다.

2. 데모와 PoC에 강한 구조  
실데이터 연동이 어려운 상황에서도 Mock 폴백과 가상 차량 시드 기능으로 동일한 제품 경험을 유지할 수 있다.

3. 운영자 중심 UX  
단순한 차량 데이터 조회가 아니라, 관리자가 즉시 판단하고 조치할 수 있는 관제 흐름으로 설계되어 있다.

4. 확장 가능한 구조  
현재는 테슬라 단일 차종 MVP지만, provider 구조를 통해 멀티 OEM 확장 가능성을 확보했다.

5. 기술적 진입 장벽 경험 확보  
Tesla OAuth, 한국 `na` 리전 대응, Partner Register, Sync 파이프라인, 감사 로그 구조를 직접 구현하며 진입 장벽이 높은 영역을 먼저 통과했다.

## 8. 핵심 지표 (Traction)
현재는 매출과 고객 수보다 "기술 검증"과 "제품 완성도"가 핵심 지표다.

### 현재까지 확보한 성과
- Tesla OAuth 연동 구현 완료
- Tesla Partner Register 완료, `412` 오류 해결
- Supabase PostgreSQL 전환 및 Vercel 배포 검증 완료
- `GET /api/vehicles` 로컬/배포 200 확인
- 실 Tesla 차량 연동 검증 완료
- Mock 12대 시드 및 대시보드 시연 가능
- 가상 차량 추가 기능 구현 완료
- 차량 목록, 대시보드, 지도, 차량 상세, 설정 화면 구현 완료
- AuditLog / ApiCallLog 구현 완료
- 브랜딩 `보리차` 적용 완료

### 제품 지표 관점의 의미
- 동시 표시 가능 차량 수: 최소 10대 이상 시연 가능
- 차량 위치/상태 갱신 주기: 5분 이내(폴링 기준)
- 실데이터 ↔ Mock 전환: 30초 이내 전환 가능 구조 검증
- 데모 시나리오 3종 무중단 수행 목표 정의

### 다음 단계에서 추적할 지표
- PoC 고객 수
- 월간 활성 법인 수
- 연동 차량 수
- 차량당 월간 데이터 갱신 수
- 이상 상태 탐지 대응 시간
- Mock 폴백 없이 안정 동작하는 차량 비율

## 9. 팀 소개
### Founder-Market Fit
- 15년차 백엔드 개발 경력을 기반으로 제품 아키텍처, 데이터 모델, API 연동, 운영 로깅 구조를 빠르게 설계·구현할 수 있다.
- 1인 개발 + AI 협업 방식으로 짧은 기간 안에 MVP를 빠르게 검증하고 있다.
- 복잡한 기능보다 MVP 핵심 기능을 우선 구현하고, 문서화와 검증을 병행하는 실행 중심 방식이 강점이다.

### 팀 역량
- 개발: 백엔드 중심 시스템 설계, 데이터 모델링, API 연동, 배포, 운영 로그 설계
- 기획: 요구사항 문서화, 단계별 체크리스트 기반 제품 우선순위 관리
- 사업화 보완: 기획, 마케팅, SNS 홍보 역량을 연결해 초기 시장 검증과 사용자 커뮤니케이션까지 확장 가능

### 팀의 강점
- 구현 속도와 구조적 사고를 동시에 가져가는 팀
- 문제 정의부터 데모 안정화까지 직접 해결하는 실행형 팀
- 투자 이후 멀티 OEM, Telemetry, 운영 자동화로 확장 가능한 기반을 이미 확보한 팀
