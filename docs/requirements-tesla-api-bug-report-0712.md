(내가 생각한 원인 분석)
주차모드에서 차량이 깨어날때
rest 1회만 호출(정책)하여 차량 데이터 받아오는 로직 때문에
그 이후 차량이 이동중에도 그당시 시점 데이터가 그대로 노출되는 혼동을 주는것 같음

telemetry 데이터는 실시간으로 업데이트 됨

인근 충전소도 출발지 기준으로 그대로 표시되어 있음

(내가 생각한 해결 방법)
1. Telemetry 데이터는 실시간으로 들어오고 있으므로 Telemetry 패킷 내부의 도어/트렁크 관련 필드 DB를 업데이트
Telemetry가 던지는 패킷의 도어 관련 필드명이 달라서 이 부분을 매핑하지 않고 방치했을 가능성 있음
도어 관련 필드 이외에도 Telemetry 실시간 데이터로 업데이트 가능한 필드 조사.

2. 깨어나는 순간의 '스냅샷 호출 타이밍' 보정
사람이 차 문을 열고 타서 시동을 걸고 출발하기까지는 보통 30초에서 2분 정도의 시간이 걸립니다. 문을 열자마자 ONLINE 웹훅이 치고 들어왔을 때 0.1초 만에 vehicle_data API를 찔러버리면, 문이 열려있는 상태의 데이터가 수집되는 것이 당연
차가 완전히 깨어나서 기어를 드라이브(D)로 바꾸거나 안정화된 시점(약 30초~1분 후)에 Fleet API 스냅샷을 받아오면, 문이 모두 닫힌 정상적인 주행 초기 스냅샷을 확보

3. 인근 충전소 표시 문제
Telemetry GPS 좌표가 출발지에서 일정 거리 이상 벗어나면 인근 충전소 리스트 충전소 정보를 지운다.
차량이 목적지에 도착해 다시 주차할 때 갱신하여 인근 충전소를 표시해준다.
